## Errors Encountered & Resolved

### Error 8: stopSpinner가 에이전트 헤더를 지움
- **Symptom**: 박스 상단이 에이전트 헤더 위치에 그려짐
- **Root Cause**: `spinner.stop()` 후 `\x1b[A\x1b[K`가 스피너 라인이 아닌 한 줄 위(에이전트 헤더)를 지움
- **Fix**: `\x1b[A\x1b[K` 제거 — ora의 stop()이 이미 스피너 라인 정리
- **Commit**: `409efc9`

### Error 9: 박스 왼쪽 상단 코너 깨짐
- **Symptom**: `╭` 코너가 스피너 잔여물과 겹쳐서 깨져 보임
- **Root Cause**: ora가 stderr로 출력, 박스는 stdout — 서로 다른 스트림 간 커서 위치 불일치
- **Fix**: ora에 `stream: process.stdout` 추가, `stopSpinner()`에 `\r\x1b[K` 추가
- **Commit**: `d0ffc43`

### Error 10: 텍스트가 박스 밖으로 넘침
- **Symptom**: 긴 텍스트가 터미널 줄바꿈 시 `│` 보더 없이 출력
- **Root Cause**: 터미널 자동 줄바꿈은 보더 프리픽스를 추가하지 않음
- **Fix**: `wrapText()` 함수로 `columns - 4` 폭에 맞게 단어 단위 줄바꿈
- **Commit**: `b3722ae`

## Decisions Made

### Decision 21: Markdown 라인 버퍼링 방식
- **Choice**: 라인 단위 버퍼링 후 마크다운 변환 (문자 단위 X)
- **Rationale**: 마크다운 패턴(`**bold**`)이 완성된 후 처리해야 정확. 라인 단위 지연은 체감 불가.

### Decision 22: Word Wrap은 raw 텍스트 기준
- **Choice**: 마크다운 렌더링 전 raw 텍스트 길이로 wrap
- **Rationale**: 보수적(마크다운 마커 제거 후 더 짧아지므로 overflow 불가). ANSI 코드 폭 계산보다 단순.

### Decision 23: ora 스트림을 stdout로 통일
- **Choice**: ora spinner에 `stream: process.stdout` 명시
- **Rationale**: 박스 렌더링도 stdout → 같은 스트림으로 커서 위치 일관성 확보.
